# Remote Repositories Notes

### What is a remote repository?
A remote repository is a version of your project that is hosted on the internet or a network somewhere (like GitHub). Unlike your local repository which lives on your phone or computer, a remote repository allows you to back up your history and collaborate with others.

### What is origin?
In Git, origin is the default shorthand name given to the remote repository URL that your local repository is linked to. Instead of typing the full HTTPS URL every time you push or pull, you just use the word origin.

### Why do developers use GitHub?
Developers use GitHub for cloud backup of their code, version tracking visibility, open-source project contributions, team collaboration, and code review management.

### Difference between local and remote repositories
* Local Repository: Lives entirely on your physical machine. You manage files in the working directory, stage changes, and create local commits here without needing an internet connection.
* Remote Repository: Lives on a remote server (GitHub). It acts as a central hub where local commits are pushed to share with teammates or pull down updates to stay synchronized.
