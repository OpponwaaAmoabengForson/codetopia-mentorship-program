# Week 1 Capstone Reflection

### 1. What are the 3 Git states?
* Modified: Changes have been made to a file in the working directory, but it hasn't been staged or committed yet.
* Staged: The modified file has been marked in its current version to go into the next commit snapshot.
* Committed: The changes are safely stored in the local Git repository database.

### 2. Why is Git useful?
Git acts as a version control time machine. It tracks history, provides a safety net to roll back mistakes, and allows developers to isolate new features without breaking the working code.

### 3. Why do commit messages matter?
Commit messages provide context on *why* changes were made. Keeping them short, professional, and in the imperative tense makes the project history clean and easy for mentors or teammates to read.

I learned to configure Git credentials, initialize repositories, and navigate the total workflow.
